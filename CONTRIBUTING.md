# Contributing

Thanks for contributing to ProCentricEPG.

## Development Focus Areas

- Additional EPG data sources
- Support for other regions
- Enhanced error handling
- Performance optimizations

## Branch and PR Workflow

1. Create a feature branch from `main`.
2. Make changes and open a PR back to `main`.
3. Use a conventional PR title (required by CI), for example:
   - `feat(epg-generator): add ...`
   - `fix(epg-admin): resolve ...`
   - `chore(ci): update ...`
4. Ensure required checks pass:
   - `Lint / ruff`
   - `Conventional Commits / validate-pr-title`
5. Merge using squash merge (recommended).

## Conventional Commits and Versioning

Release Please manages semantic versioning separately for:
- `epg_generator` (`epg-generator-vX.Y.Z`)
- `epg_admin` (`epg-admin-vX.Y.Z`)

Version bump behavior is based on conventional commit semantics:
- `fix:` -> patch
- `feat:` -> minor
- `!` or `BREAKING CHANGE:` -> major

Important:
- `chore:` usually does not trigger a normal version bump.
- The release PR title (`chore(main): release ...`) is generated by automation and does not itself imply a major bump.

## Release and Container Build Flow

Container publishing and releases are related but not strictly dependent.

After PR merge to `main`:
1. `containers.yml` runs on push to `main` and publishes:
   - build-channel tags (`edge`)
   - `sha-<shortsha>`
2. `release-please.yml` updates/opens release PR(s) for changed component(s).
3. When release PR(s) are merged, tags are created:
   - `epg-generator-vX.Y.Z`
   - `epg-admin-vX.Y.Z`
4. `release-please.yml` also builds and publishes release images for released components:
   - `:<version>`
   - `:latest`

So:
- You get continuously updated build-channel images from `main`.
- You get immutable versioned images when release tags are created.

## Container Registries and Deployment Defaults

- Docker Compose defaults to GHCR images.
- Kubernetes base manifests default to GHCR images.
- Local source builds remain available using `docker-compose.build.yml` or Kustomize image overrides.

## Local Checks

Before opening a PR, run lint locally:

```bash
ruff check epg_generator/src epg_admin/src
```
